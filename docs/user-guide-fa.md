# راهنمای استفاده (ویژه حسابدار)

این نرم‌افزار یک سیستم مالی + انبارداری چندشرکتی (Multi‑Tenant) است. هر «شرکت» یک دیتابیس مستقل دارد و شما با هدر `x-tenant-id` مشخص می‌کنید روی کدام شرکت کار می‌کنید.

## 1) مفاهیم کلیدی

### شرکت (Tenant)
هر شرکت:
- مشتری‌ها/تامین‌کننده‌ها/کالاها/فاکتورها/اسناد و… را جداگانه نگه می‌دارد.
- کدینگ حساب‌ها (Chart of Accounts) و اسناد حسابداری مخصوص خودش را دارد.

### انبار و موجودی
موجودی به شکل زیر مدیریت می‌شود:
- `StockItem`: موجودی فعلی هر کالا در هر انبار
- `InventoryTxn`: تراکنش‌های ورود/خروج/انتقال/تعدیل

### روش ارزش‌گذاری موجودی (AVG)
برای ارزش موجودی از **میانگین موزون (Average Cost)** استفاده می‌کنیم:
- در ورود کالا (Purchase/IN) اگر `unitCost` داده شود، `avgCost` به‌روز می‌شود.
- ارزش موجودی = `qty * avgCost`

### حسابداری دوطرفه (Double Entry)
سیستم از سند حسابداری و ردیف‌های بدهکار/بستانکار استفاده می‌کند:
- `JournalEntry` (سند)
- `JournalLine` (ردیف)

ثبت اتوماتیک (فعلاً ساده) انجام می‌شود:
- فروش: Dr دریافتنی / Cr فروش / Cr VAT فروش
- خرید: Dr موجودی کالا / Dr VAT خرید / Cr پرداختنی
- بهای تمام‌شده فروش (COGS): Dr بهای تمام‌شده / Cr موجودی کالا (بر اساس AVG)
- پرداخت‌ها: دریافت/پرداخت/انتقال

## 2) جریان کاری پیشنهادی (روزمره)

### گام A — راه‌اندازی اولیه شرکت
1) ساخت شرکت جدید
2) بررسی کدینگ حساب‌ها (کدهای پیش‌فرض ساخته می‌شوند)
3) تعریف انبارها (حداقل یک «انبار اصلی» ساخته می‌شود)
4) تعریف واحدها/کالاها

### گام B — ثبت خرید (ورود کالا)
1) ثبت تامین‌کننده
2) ثبت فاکتور خرید (Purchase Invoice) با آیتم‌ها
3) سیستم به‌صورت اتومات:
   - موجودی را وارد انبار می‌کند (IN)
   - میانگین بهای تمام‌شده `avgCost` را به‌روزرسانی می‌کند (اگر unitCost داشته باشید)
   - سند حسابداری خرید ایجاد می‌کند

### گام C — ثبت فروش (خروج کالا)
1) ثبت مشتری
2) ثبت فاکتور فروش (Sales Invoice) با آیتم‌ها
3) سیستم به‌صورت اتومات:
   - از انبار خروج می‌زند (OUT)
   - سند فروش را می‌زند
   - سند بهای تمام‌شده فروش را می‌زند (COGS) با روش AVG

### گام D — دریافت/پرداخت
1) ثبت پرداخت:
   - برای فروش: Payment با direction=IN و referenceType=SalesInvoice
   - برای خرید: Payment با direction=OUT و referenceType=PurchaseInvoice
2) سیستم به‌صورت اتومات:
   - سند پرداخت ایجاد می‌کند
   - وضعیت فاکتور را به‌روزرسانی می‌کند: Unpaid / Partially / Paid

## 3) گزارش‌ها

### تراز آزمایشی (Trial Balance)
`GET /api/v1/accounting/trial-balance`

### سود و زیان (Profit & Loss)
`GET /api/v1/reports/profit-loss?from=YYYY-MM-DD&to=YYYY-MM-DD`

### ترازنامه (Balance Sheet)
`GET /api/v1/reports/balance-sheet?at=YYYY-MM-DD`

### ارزش موجودی (Inventory Valuation)
`GET /api/v1/reports/inventory-valuation`

### دفترکل/گردش حساب (Ledger)
`GET /api/v1/reports/ledger?accountId=...&from=...&to=...`

## 4) کار با چند شرکت (Tenant)
برای تمام مسیرهای tenant (مثل customers/products/sales/...) باید:
- توکن JWT داشته باشید (Authorization: Bearer ...)
- هدر `x-tenant-id` را ارسال کنید.

برای گرفتن tenantId:
`GET /api/v1/tenants/mine`

## 5) اجرای سیستم با Docker (روی سیستم شما)

### پیش‌نیازها
- Docker + Docker Compose
- Node.js (فقط برای اجرای `pnpm docker:up` لازم است)

### مراحل
1) ساخت env:
```bash
cp .env.example .env
```

2) اجرای سرویس‌ها:
```bash
pnpm docker:up
```

3) آدرس‌ها:
- پنل: `http://localhost:3000`
- API: `http://localhost:3001/api/v1`

### نکته درباره دیتابیس‌ها
وقتی یک tenant می‌سازید، سیستم یک دیتابیس جدید با نام `tenant_<slug>` ایجاد می‌کند و schema را اعمال می‌کند.
